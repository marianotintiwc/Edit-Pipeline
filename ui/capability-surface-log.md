# Capability Surface Log — Node-and-Link Creative Freedom

Generated by Overnight Guardian Mode v2. Priorities: P0 (must-have for MVP), P1 (high value), P2 (post-MVP).

---

## 1. Node graph concept for UGC

### Input nodes

- **Clips:** Ordered list of scene/broll/endcard/introcard with URL, trim, optional alpha_fill
- **Music:** URL (null | random | explicit), volume, loop
- **Geo:** Market (MLA, MLB, MLC, MLM)

### Processing nodes

- **Plan:** request_text, plan_only; infers subtitle/music/interpolation/horizontal/vertical
- **Style:** style_overrides (font, color, highlight, animation, shadow); structured controls vs raw JSON
- **Alpha:** broll_alpha_fill, endcard_alpha_fill, introcard_alpha_fill; blur, chroma key, levels
- **Subtitles:** subtitle_mode (auto | manual | none), manual_srt_url, transcription tuning
- **Postprocess:** frame interpolation, color grading, grain, vignette, halation

### Output node

- **Render:** Submit job; output_filename, output_folder, output_bucket

## 3. Hybrid entry: Simple mode vs Graph mode

- **Simple mode:** Current forms (Brief → Style → Review). Fastest path; one primary CTA per step.
- **Graph mode:** Node canvas for power users; drag nodes, connect inputs→processing→output.
- **Toggle:** One-click switch (ComfyUI App View pattern); graph serializes to same API payload as forms.
- **Backend:** No API changes; graph compiles to existing JobInput/request schema.

## 4. Implementation sketch

| Piece | Recommendation |
|-------|----------------|
| Library | React Flow / Xyflow (`@xyflow/react`) |
| Custom nodes | React components; `nodeTypes` map; handles for inputs/outputs |
| Layout | ElkJS or Dagre for auto-layout |
| Serialization | nodes + edges → `JobInput`; bidirectional sync with form state |
| Canvas Flow alignment | Use semantic tokens (--bg-primary, --accent-*); staged-flow mental model (Brief→Style→Review = pipeline) |

---

## 5. Canvas Flow alignment

- **Semantic tokens:** `--bg-primary`, `--text-primary`, `--accent-primary`, `--duration-*`, `--ease-*`
- **Staged flow:** Resize → Translate → Iterate maps to Input → Processing → Output node model
- **Batch utility:** Upload → format → generate → review; same pattern as Batch Import → Preview → Submit

## 6. Priorities

### P0 (must-have for graph MVP)

- Input nodes: Clips, Music, Geo
- Processing nodes: Plan (minimal), Style (basic style_overrides subset)
- Output node: Render
- Serialization to existing API payload
- Simple mode / Graph mode toggle

### P1 (high value)

- Alpha node with guided controls (blur, chroma key)
- Subtitles node
- Postprocess node (interpolation, color grading)
- Live preview in nodes (run plan_only per subgraph)
- Minimap, undo/redo

### P2 (post-MVP)

- Reusable templates (save/load graph)
- Subflows / grouped nodes
- Batch: one graph, many inputs
- AI-suggested node connections from request_text
